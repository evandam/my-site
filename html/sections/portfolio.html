An Android game of No-Limit Texas Hold'em, featuring a strong AI opponent.

The AI uses theories used in Poker including: pot odds, hand strength, and deception. The opponent is also configurable to simulate different play styles. This project was one of my first attempts of creating a multi-threaded application, with the standard UI and main thread of Android, and the addition of a thread for the AI to perform it's calculations.

A Java application to solve a Rubik's Cube in the fewest possible moves, and a WebGL application to display the solution.

The solver used IDA* Search as defined by Korf . This project posed countless challenges to implement an efficient algorithm in both time and space.

The project to create an animated and interactive WebGL application gave insight into the low-level workings and concepts of computer graphics.

A Chess-playing agent, and an accompanying WebGL visualizer of the games.

The agent implemented a Minimax adversarial search with Alpha-Beta pruning to look ahead 6 moves, while keeping the clock in mind. The game interacted with other players over a RESTful web server. There were many interested challenges, including finding an accurate heuristic to represent the strength of making a move.

The WebGL application used three.js and dat-gui to handle user interaction. By polling the web server, games were played out in real-time. This project took a deep dive into computer graphics concepts like using 3D models, camera movement, and even some particle effects. Unfortunately, I no longer have this project to display.

A JavaScript simulation of an operating system, using the HTML5 canvas.

This project included emulating a CPU in JavaScript and implementing an instruction set, writing device drivers, a file system, and using various operating-system algorithms.

The OS features an interactive shell, multiple process-scheduling algorith, memory swapping, and a persistent file system sitting on a virtual hard drive based on HTML5 localStorage.
